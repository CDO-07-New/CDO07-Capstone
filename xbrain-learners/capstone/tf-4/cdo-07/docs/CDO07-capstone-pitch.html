Service Selection #1 · Compute

AI Engine: ECS Fargate vs Lambda vs EKS

AI Serving cần long-running, stateful, predictable latency — 3 yêu cầu loại bỏ Lambda và EKS ngay từ đầu.

Tiêu chí✅ ECS Fargate (Đã chọn)LambdaEKS (Kubernetes)
Long-running ≥ 2h✅ Không giới hạn❌ Timeout 15 phút — block cửa sổ test ≥ 2h✅ Không giới hạn
Cold Start & Latency✅ Không cold start · p99 < 200ms❌ Cold start 500ms+ với ML libraries✅ Không cold start
Stateful Circuit Breaker✅ State sống trong vòng đời task❌ State reset mỗi invocation✅ Stateful
Ops Overhead✅ AWS quản lý OS, patching, host✅ Zero-ops❌ Quản lý control plane K8s phức tạp
Chi phí (2 tasks 24/7)~$67.22/tháng — predictableRẻ nếu ít invoke, nhưng không phù hợp❌ Control plane ~$73/tháng + nodes
Zero-ops requirement✅ Serverless container✅ Serverless function❌ Vi phạm — cần quản lý cluster
Kết luận: ECS Fargate = duy nhất đáp ứng cả 3 yêu cầu: long-running ≥ 2h, stateful circuit breaker, zero-ops. Spec: AI Engine 0.5 vCPU / 1GB RAM · 3 Mock Services 0.5 vCPU / 0.5GB mỗi task.

Security Design

Zero-Trust Network: Khép Kín 100%

Không có Internet Gateway. Không có NAT Gateway. 8 VPC Endpoints. Một ngoại lệ duy nhất có chủ ý.

Network Security · Zero-Trust

8 VPC Endpoints · 5 Security Groups · KMS CMK

🔒 8 VPC Endpoints

SSMSNSAMG (Grafana)ECRCloudWatchTimestreamS3 GatewayKDS

7 Interface + 1 S3 Gateway. Chi phí: ~$50.40/tháng — đánh đổi có chủ ý cho SOC2 compliance.

⚠ Ngoại lệ Duy Nhất: Lambda SNS-to-SLACK

Chạy Non-VPC Lambda → nhận SNS trigger qua AWS internal → gửi HTTPS 443 ra hooks.slack.com. Không cần NAT Gateway, không phá vỡ private boundary.

🔐 5 Security Groups

  • tf4-cdo07-alb-sg — ALB Internal (HTTPS 443 inbound)
  • tf4-cdo07-app-sg — ECS Tasks (HTTP 8080 từ ALB)
  • tf4-cdo07-lambda-sg — Lambda functions (no inbound)
  • tf4-cdo07-vpce-sg — Interface VPC Endpoints
  • tf4-cdo07-influxdb-sg — Timestream (TCP 8086)
k6 Load Test: External k6 kết nối qua AWS SSM Port Forwarding Session → SSM VPC Endpoint → Internal ALB. ALB path-based routing: /v1/predict → AI Engine · /* → Mock Services.

AI & Resilience

Predict sớm · Fail-Open khi không predict được

EWMA+STL model zero token cost. Fail-Open Fallback đảm bảo hệ thống không bao giờ im lặng.

AI Engine + Resilience

EWMA+STL Model · Fail-Open Fallback · Cost CB

🧠 AI Model: tf4-ewma-stl-v1

  • EWMA + STL Decomposition — NumPy-based
  • Zero token cost (không dùng LLM)
  • Input: 2h sliding window từ Timestream InfluxDB
  • Output: 5-part Capacity Recommendation
  • "Scale payment-gw 2→4 tasks; confidence=0.87"

🛡 Fail-Open Fallback (RTO <1s)

  • AI Engine timeout > 5s → kích hoạt tức thì
  • CloudWatch metrics qua VPC Endpoint
  • Static: CPU > 30%, Mem > 50%, Conn > 450, Queue > 10k
  • Vẫn ghi audit S3 + push annotation Grafana + alert Slack

💸 Cost Circuit Breaker

  • Alert $165 (80%): push Grafana annotation + Slack
  • Hard stop $200 (100%): Lambda CB set SSM inference_enabled = false
  • Toàn bộ inference dừng trong <5s

[Action] Scale ECS service [Target] payment-gw [From→To] 2 tasks → 4 tasks [Confidence] 0.87 [Evidence] s3://.../audit.json

SRE review Grafana → Manual Approve Gate → Action.
